初始化仓库
如果你有一个新的项目目录，想用 Git 来管理版本，可以在项目目录下打开终端或命令行，输入git init。这会在当前目录下创建一个名为.git的隐藏目录，它包含了 Git 仓库的所有元数据，如提交历史、分支信息等。例如，你有一个名为“MyProject”的文件夹，进入该文件夹后执行git init，MyProject 就变成了一个 Git 仓库。
添加文件到暂存区
当你在项目目录下创建或修改文件后，这些文件的更改不会自动被 Git 跟踪。需要使用git add命令将文件添加到暂存区。暂存区是 Git 用来存储即将提交到仓库的文件更改的地方。
如果你想添加一个特定的文件，比如file.txt，可以使用git add file.txt。如果你想添加当前目录下所有更改的文件，可以使用git add .（注意是英文句号，表示当前目录下的所有文件）。
提交更改
当文件被添加到暂存区后，就可以使用git commit命令将更改提交到仓库了。提交时需要添加一个提交信息，用来描述这次提交的内容。例如：
git commit -m "Add new feature"（这里的-m后面跟的是提交信息，描述了添加了一个新功能）
提交操作会将暂存区的更改保存到 Git 仓库的历史记录中。每次提交都会生成一个唯一的提交 ID，用来标识这次提交。
